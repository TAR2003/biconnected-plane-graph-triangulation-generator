\begin{abstract}
    We present a new algorithm that can generate all possible triangulations of biconnected plane graphs very quickly. Our method builds on previous work but solves the key problem of avoiding invalid edges (multi-edges) that previous methods couldn't handle. The algorithm runs in constant time per triangulation after a simple preprocessing step, making it practical for real applications in mesh generation, chip design, and graph visualization.
\end{abstract}

\subsection{Why is This Hard?}

\section{The Safe Root Method}

\subsection{What is a Safe Root?}

    For a face $F$ with $n \geq 4$ vertices, a vertex $v$ in $F$ is a \textbf{safe root} if we can connect $v$ to all non-adjacent vertices in $F$ without creating any parallel edges.
\end{definition}

\subsection{Finding Safe Roots: The Two-Pointer Algorithm}

The core of our method is an efficient, linear-time algorithm for identifying a safe root within a face $F$ with $n \geq 4$ vertices. The algorithm operates on the cyclic sequence of vertices $V = (v_0, v_1, v_2, \ldots, v_{n-1})$ defining the face $F$.

\textbf{Intuition:} The algorithm checks potential ``candidate'' edges from vertices on one side of the cyclic sequence to vertices on the other. It uses two pointers, \textit{left} and \textit{right}, to traverse the list of vertices. The goal is to find a vertex $v_i$ such that the ``long'' chord $(v_i, v_j)$ for a distant $j$ does not yet exist in the graph.

\textbf{Algorithm: findSafeRoot}

\textbf{Input:} A list \textit{vertices} of size $n$ (with $n \geq 4$) representing the vertices of face $F$ in cyclic order. A global hash table \textit{edgeTable} for $O(1)$ lookups of existing edges.

\textbf{Output:} The index $i$ of a safe root vertex $v_i$, or $-1$ if no safe root is found.

\textbf{Initialization:}
\begin{itemize}
    \item Let $left = 0$. This pointer advances from the start of the list.
    \item Let $right = n - 2$. This pointer retreats from the end. We use $n-2$ because the vertex at index $n-1$ is adjacent to vertex at index $0$.
\end{itemize}

\textbf{Main Loop:}
While $(right - left > 1)$:
\begin{enumerate}
    \item Let $u = vertices[left]$ and $v = vertices[right]$.
    \item Check if edge $(u, v)$ exists in $edgeTable$.
    \item If $edgeTable.contains(u, v)$ is TRUE: Move $left$ forward ($left++$).
    \item If $edgeTable.contains(u, v)$ is FALSE: Move $right$ backward ($right--$).
\end{enumerate}

\textbf{Termination:} Return $left$ when pointers are at most one index apart.

\textbf{Why this works:} Upon termination, the left pointer has converged to a vertex $v_l$ for which connecting to all non-adjacent vertices is safe. The movement ensures that if a blocking chord existed for a previous candidate, we correctly skip that candidate.

\textbf{Complexity:} $O(n)$ time with $O(1)$ hash table lookups.

Here's our algorithm illustrated step-by-step:

\begin{algorithm}
    \caption{Find Safe Root - Visual Walkthrough}
    \begin{algorithmic}[1]
        \REQUIRE Face $F$ with vertices $v_1, v_2, \ldots, v_n$ (in order around the face)
        \REQUIRE Hash table $H$ containing all existing edges
        \ENSURE A safe root vertex
        \STATE $left \leftarrow 1$ \COMMENT{Start from first vertex}
        \STATE $right \leftarrow n-1$ \COMMENT{Start from second-to-last vertex}
        \WHILE{right - left > 1} 
            \STATE $u \leftarrow v_{left}$ 
            \STATE $v \leftarrow v_{right}$ 
            \IF{edge (u, v) in H}
                \STATE $left \leftarrow left + 1$ \COMMENT{Move left pointer right}
            \ELSE
                \STATE $right \leftarrow right - 1$ \COMMENT{Move right pointer left}
            \ENDIF
        \ENDWHILE
        \STATE \textbf{return} $v_{left}$ \COMMENT{This vertex is safe to use}
    \end{algorithmic}
\end{algorithm}

\subsection{Why This Works: The Mathematical Foundation}

The correctness of our two-pointer algorithm rests on a fundamental property of planar graphs and the structure of blocking chords. We provide a rigorous analysis of why the algorithm always terminates at a safe root.

\begin{lemma}[Convergence Property]
    The two-pointer algorithm always terminates and finds a safe root in $O(n)$ steps.
\end{lemma}

\begin{proof}
    We analyze the algorithm's behavior by considering the movement of pointers and the structure of blocking chords.

    \textbf{Termination:} Each iteration either increments $left$ or decrements $right$. Since $left$ starts at $0$ and $right$ starts at $n-2$, and we stop when $right - left \leq 1$, the algorithm terminates in at most $n-2$ steps.

    \textbf{Correctness:} We prove that the final value of $left$ points to a safe root. Consider the final state where $left = k$ and the algorithm terminates.

    \textit{Key Insight:} The algorithm's pointer movements encode information about the planarity structure. When we encounter a blocking chord $(v_l, v_r)$, moving $left$ forward means we are ``avoiding'' the vertex that would create multi-edges.

    Let $r = v_k$ be the vertex at position $left$ when the algorithm terminates. We claim that $r$ is safe for triangulation.

    \textit{Case Analysis:} Suppose, for contradiction, that $r$ is not safe. Then there exists some non-adjacent vertex $v_j$ such that edge $(r, v_j)$ already exists in the graph.

    During the algorithm's execution, there must have been a step where $left = k$ and $right = j$ (or the algorithm would have detected this blocking chord). At that step, the algorithm would have found $(v_k, v_j)$ exists and moved $left$ forward, contradicting the fact that the algorithm terminated with $left = k$.

    Therefore, no such blocking chord exists for $r$, making it a safe root. \qed
\end{proof}

The key insight is that planarity constraints create natural ``barriers'' that limit where problematic edges can exist. The algorithm exploits this structure to efficiently navigate to a safe starting point.

\begin{theorem}[Safe Root Existence]
    Every face with 4 or more vertices has at least one safe root.
\end{theorem}

\begin{proof}
    Let the vertices of the face $F$ in cyclic order be $v_0, v_1, v_2, \ldots, v_{n-1}$ where $n \geq 4$. A vertex $v_i$ is a safe root if, for every vertex $v_j$ non-adjacent to $v_i$ in $F$, the edge $(v_i, v_j)$ is not already present in the graph $G$.

    We prove the existence of a safe root by contradiction, leveraging the planarity and biconnectivity of $G$.

    \textbf{1. Assumption for Contradiction:}
    Assume, for the sake of contradiction, that no vertex in $F$ is a safe root. This means that for every vertex $v_i \in F$, there exists at least one non-adjacent vertex $v_j$ such that the edge $(v_i, v_j)$ already exists in $G$.

    \textbf{2. Identifying Problematic Edges:}
    For each vertex $v_i$, identify one such "problematic" non-adjacent vertex $v_{k(i)}$ such that the edge $e_i = (v_i, v_{k(i)})$ exists in $G$. Since $v_i$ and $v_{k(i)}$ are non-adjacent in $F$, the edge $e_i$ must be a chord of the face $F$.

    \textbf{3. Analyzing the Implied Structure:}
    Consider the chord $e_0 = (v_0, v_j)$ for the starting vertex $v_0$ (where $j \notin \{n-1, 1\}$). This chord divides the face $F$ into two regions: $R_1$ (containing vertices $v_1, \ldots, v_{j-1}$) and $R_2$ (containing vertices $v_{j+1}, \ldots, v_{n-1}$).

    \textbf{4. The Contradiction:}
    Consider a vertex $v_r$ in $R_2$. By our assumption, it must have a chord $e_r = (v_r, v_s)$. If $v_s$ is in $R_1$, then the edge $e_r$ would cross the chord $e_0$, violating planarity. Therefore, all chords incident to vertices in $R_2$ must connect to other vertices in $R_2$.

    This means the chord $e_0 = (v_0, v_j)$ acts as a separating pair. If we remove vertices $v_0$ and $v_j$, the graph becomes disconnected: vertices in $R_2$ are isolated from vertices in $R_1$ and the rest of the graph. This contradicts the biconnectivity of $G$.

    \textbf{5. Conclusion:}
    Our assumption leads to a violation of biconnectivity. Therefore, there must exist at least one safe root in face $F$. \qed
\end{proof}

\subsection{Safe Root Algorithm Implementation}

\begin{verbatim}
int Face::findSafeRoot() {
    int n = vertices.size();
    if (n < 4) return -1;  // Cannot triangulate
    
    int left = 0, right = n - 2;
    
    while (right - left > 1) {
        int u = vertices[left];
        int v = vertices[right];
        
        if (edgeTable.contains(u, v)) {
            left++;  // Edge exists, try next left vertex
        } else {
            right--; // Edge safe, try next right vertex  
        }
    }
    
    return left;  // Safe root found
}
\end{verbatim}

\section{Theorems on Algorithm Properties}

\begin{theorem}[Persistence of Blocking Chords]
    Let $G$ be a biconnected plane graph, and let $T$ be a triangulation of $G$ generated by our algorithm. Let $e = (a,b)$ be a blocking chord in $T$---an edge discovered during the safe root finding process that prevents a flip operation. If $T'$ is any descendant of $T$ in the enumeration tree, then the chord $e$ is also present in $T'$. Furthermore, $e$ will never be flipped or removed in the lineage from $T$ to $T'$.
\end{theorem}

\begin{proof}
    We prove this by analyzing the algorithmic construction and the irrevocable nature of decisions in our enumeration process.

    \textbf{1. The Enumeration Tree as a Decision Tree:}
    The algorithm generates triangulations by recursively adding chords to faces. Each node represents a partially triangulated graph. The path from root to leaf represents the sequence of chords added to reach that triangulation.

    \textbf{2. The Blocking Chord Event:}
    Consider a face $F$ with vertices in cyclic order. During the execution of findSafeRoot, the two-pointer algorithm initializes $left = 0$ and $right = n-2$, representing vertices $v_l$ and $v_r$. The critical event occurs when the algorithm finds that chord $e = (a,b) = (v_l, v_r)$ already exists in the graph. This causes the left pointer to advance: $left++$.

    Let $r$ be the eventual safe root found, and let $c$ be any vertex non-adjacent to $r$ in $F$. The presence of blocking chord $(a,b)$ fundamentally constrains the triangulation process.

    \textbf{3. Impossibility of Flip in Descendants:}
    Consider any descendant triangulation $T'$ of $T$. For the flip of $(a,b)$ to become possible in $T'$, the quadrilateral $(a,c,b,d)$ must form a face where the flip would replace $(a,b)$ with $(c,d)$. However:

    \begin{itemize}
        \item The chord $(c,d)$ already exists in the graph (this is why $(a,b)$ was blocking).
        \item The genealogical tree construction only adds edges, never removes them.
        \item Therefore, $(c,d)$ remains present in every descendant $T'$.
    \end{itemize}

    The flip operation is only defined for creating a new edge, not for creating a multi-edge. Since $(c,d)$ already exists, flipping $(a,b)$ would create a parallel edge, which is invalid in simple graphs.

    \textbf{4. Ancestral Decisions Constrain Descendants:}
    The chord $e$ is an ancestral decision for all triangulations in the subtree rooted at $T$. The safe root $r$ was chosen precisely because it could connect to vertex $c$ without creating multi-edges, but the blocking chord $(a,b)$ demonstrates that alternative connection patterns are forbidden.

    Once face $F$ is subdivided by adding chords from $r$, these decisions create permanent separators. The algorithm generates all triangulations of the resulting sub-faces independently, but the blocking constraint persists throughout.

    \textbf{5. Formal Inductive Argument:}

    \textit{Base Case:} In $T$, the flip of $(a,b)$ is invalid because $(c,d)$ exists.

    \textit{Inductive Step:} Assume that in descendant $T_k$, the flip is still invalid. When adding a new chord to form $T_{k+1}$, the existing chord $(c,d)$ cannot be removed---only new chords are added. Therefore, the obstruction remains, and the flip of $(a,b)$ stays invalid in $T_{k+1}$.

    \textbf{Conclusion:} By induction, chord $(a,b)$ cannot be flipped in any descendant of $T$. Since our algorithm is based on adding edges (not flipping them), $e$ remains permanently fixed in all descendants. The blocking property is inherited throughout the entire subtree. \qed
\end{proof}

\begin{theorem}[Completeness Preservation]
    The pruning strategy based on multi-edge detection does not eliminate any valid triangulations from the genealogical tree.
\end{theorem}

\begin{proof}
    This follows directly from Theorem 2. If a triangulation $T$ contains a blocking chord whose flip would create a multi-edge, then this chord can never be flipped in any descendant of $T$. Consequently, pruning the entire subtree rooted at $T$ is correct---it contains no valid triangulations that are not reachable through other branches of the enumeration tree.

    The key insight is that our algorithm explores all valid triangulation paths. When a blocking chord prevents a certain triangulation from being generated in one branch, the same triangulation (if valid) must be reachable through a different sequence of choices in another branch where that blocking condition does not arise. \qed
\end{proof}
